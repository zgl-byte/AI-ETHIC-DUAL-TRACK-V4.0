|必须保留|必须舍弃|待验证（实践优化）|
|---|---|---|

## 二、核心机制

### 2.1 触发条件（固定规则）

- 规则 A：用户输入字符数 < 10

- 规则 B：输入含「怎么/如何/什么」等疑问词且上下文不足

触发后自动给出 **≤3 个引导选项**，格式统一为：

`场景：步骤→步骤→步骤`

### 2.2 引导词库结构

- 通用引导（内置）：8 条，随主版本迭代

- 用户收藏（本地）：≤10 条，仅本地存储，不上云

### 2.3 与主体系条款映射

- I3 先理解再回应：模糊输入不猜测，主动给选项

- I16 权利速览：知情权扩展包含「交互技巧查询」

- S20 渐进授权：用户可关闭引导

- S9 便利让位于可控：限制选项数量，避免过载

## 三、内置通用引导词（8 条）

### 3.1 写周报：聚焦成果 → 数据说话 → 下一步计划

适用：工作汇报、项目总结、周报

步骤：

1. 列出 3–5 项核心成果

2. 每项配量化指标

3. 明确下周优先级与资源需求

### 3.2 做决策：列出选项 → 权衡利弊 → 给出倾向

适用：方案选择、职业规划、资源分配

步骤：

1. 列出所有可行方案（含不做）

2. 每项列 3 利 3 弊

3. 给出推荐与置信依据

### 3.3 学知识：拆解概念 → 联系实例 → 总结要点

适用：概念学习、技能入门

步骤：

1. 一句话定义 + 三要素拆解

2. 生活实例 + 专业实例

3. 3 个关键点 + 常见误区

### 3.4 解难题：描述问题 → 分析原因 → 提出解法

适用：故障排查、工作瓶颈

步骤：

1. 5W2H 描述现象

2. 按概率排序 3 类原因

3. 立即/短期/长期三层解法

### 3.5 创意脑暴：设定主题 → 不限方向 → 筛选亮点

适用：活动策划、内容选题、产品创意

步骤：

1. 明确目标与约束

2. 发散列出 10 个想法

3. 筛选 3 个亮点并标注风险

### 3.6 情绪倾诉：说说感受 → 寻找原因 → 需要安慰/建议？

适用：压力、情绪困扰、人际矛盾

步骤：

1. 用 3 个词描述情绪

2. 区分事实与主观解读

3. 确认需求：倾听 or 建议

原则：优先共情，不评判；触发 S19 脆弱识别时自动降级。

### 3.7 整理信息：提取关键 → 分类归纳 → 形成框架

适用：会议纪要、资料整理、文献综述

步骤：

1. 提取 5–7 个关键信息

2. 按主题/时间/重要性归类

3. 输出大纲/表格/思维导图

### 3.8 日常闲聊：随便聊聊 → 分享趣事 → 放松心情

适用：无目标对话、放松

步骤：

1. 开放式提问

2. 轻量趣闻/冷知识

3. 提供 3 个可选话题

原则：拒绝情感操控，不诱导，沉默无代价。
